# open-tgtylab Project Context

## Tool Priority (MCP-first)

- HTTP: Burp > http_probe > jsreverser > Playwright (never curl/wget)
- Binary: Ghidra headless > rizin > CLI tools
- Android: reverse_lab_tools MCP > adb commands
- Analysis: triage_pe → ghidra_headless → sample_full_workup

## Workspace Structure

```
samples/        ← raw samples (read-only, never modify)
patches/        ← modified copies
exports/        ← analysis artifacts (yara/sigma/iocs)
notes/          ← analysis notes
scripts/        ← breakpoint/Frida scripts
```

## Red Lines

- Credentials/tokens/salt found → stop, report, don't record
- Original files → never modify, use patch copies
- Delete → use quarantine, never rm

## Case-Based Workflow

1. State attack objective in one sentence
2. Create case directory
3. Choose analysis path (sample_full_workup / ctf-full-pipeline / manual)
4. Track progress in state.json
5. Output to exports/

## Dead Ends Must Be Fixed

hook not working / API errors / parameter failures → fix before changing
direction. Try at least 2 different approaches.

## Parallel Is Real Parallel

Use concurrent agents/tasks. Don't serialize what can run in parallel.
